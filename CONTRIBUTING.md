# Contributing to JobsGov

We welcome contributions! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/JobsGov.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Follow the setup instructions in SETUP.md

## Code Style

- Use ES6+ syntax
- 2-space indentation
- Meaningful variable/function names
- Add comments for complex logic

## Making Changes

### Backend Changes
- Follow Express.js best practices
- Add error handling
- Include JSDoc comments
- Test API endpoints

### Frontend Changes
- Keep components modular
- Use functional components with hooks
- Add PropTypes or TypeScript
- Ensure responsive design

## Testing

```bash
# Backend tests
cd backend
npm test

# Frontend tests
cd frontend
npm test
```

## Commit Guidelines

Use clear, descriptive commit messages:

```
feat: add new job matching algorithm
fix: resolve database connection issue
docs: update setup instructions
refactor: improve agent performance
```

## Pull Request Process

1. Update documentation for new features
2. Add tests for new functionality
3. Ensure all tests pass
4. Submit PR with clear description
5. Link related issues

## Areas for Contribution

- 🐛 Bug fixes
- ✨ New features
- 📝 Documentation
- 🎨 UI improvements
- ⚡ Performance optimization
- 🔒 Security enhancements

## Questions?

Open an issue with the `question` label or discuss in GitHub Discussions.

## Code of Conduct

Be respectful and inclusive. No harassment or discrimination.

Thank you for contributing! 🙏
